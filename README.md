## âš™ï¸ How It Works

### Decryption Process
1. Each script reads `ciphertext.txt` from the root directory
2. Uses keywords from `keywords.txt` as keys or to create custom alphabets
3. Attempts decryption with all possible key combinations
4. Outputs results to `decrypted_output.txt` in the script's directory

### Ranking and Filtering
The `sort_output.py` script:
- Reads `decrypted_output.txt` from the specified folder
- Scores each line based on English word ratio
- Filters out non-English text using language detection
- Ranks results by legibility score
- Outputs top candidates to `sorted_output.txt`

### Scoring Criteria
- **Word Ratio**: Percentage of words that match the English dictionary
- **Language Detection**: Uses `langdetect` to verify English text
- **Minimum Requirements**: Lines must have at least 4 words and word ratio > 0.3

### Adding New Ciphers
1. Create a new directory: `new-cipher-decryptor/`
2. Create `new_cipher.py` following the pattern of existing scripts

Edit `sort_output.py` to adjust:
- `word_ratio` threshold (currently 0.3)
- Minimum word count (currently 4)
- Language detection settings
- Output format

### File Configuration
- **keywords.txt**: Add/remove keywords for decryption attempts
- **dictionary.txt**: Modify English word list for scoring
- **ciphertext.txt**: Replace with your encrypted text

## ğŸ”§ Troubleshooting

### Common Issues
1. **File not found errors**: Ensure `keywords.txt`, `dictionary.txt`, and `ciphertext.txt` exist in root directory
2. **Import errors**: Install required packages with `pip install langdetect`
3. **Empty results**: Check that ciphertext is properly formatted and keywords are appropriate
4. **Poor ranking**: Adjust scoring thresholds in `sort_output.py`

### Performance Tips
- Use smaller keyword lists for faster processing
- Run multiple cipher types in parallel
- Monitor disk space for large output files
